🔹 ТЕМА 1/4 · DATA CENTER / RACK-SCALE
NVIDIA размещает 256 Vera CPU в одной жидкостно-охлаждаемой стойке

📌 ФАКТ
NVIDIA представила конфигурацию Vera Rubin rack-scale, объединяющую 256 CPU Vera в одной стойке с прямым жидкостным охлаждением (The Register, 16 марта 2026). Архитектура использует NVLink-домен для когерентного соединения CPU и GPU в пределах стойки.

📍 КОНТЕКСТ
Предыдущее поколение GB200 NVL72 содержало 36 Grace CPU + 72 Blackwell GPU на стойку. Переход к 256 CPU на стойку означает кратное увеличение плотности скалярного домена внутри NVLink-фабрики. Это смещает фокус с «GPU-центричной» компоновки к симметричной CPU+GPU-стойке с единым когерентным адресным пространством.

🎯 РЕЛЕВАНТНОСТЬ ДЛЯ AI DISTRIBUTION PLATFORM
Высокая. Влияет одновременно на BOM-генератор (новая SKU rack-scale), граф совместимости (CPU/GPU/NVLink-switch ревизии), TCO (питание/охлаждение на стойку растёт нелинейно).

🔮 ПРОГНОЗ НА 12 МЕСЯЦЕВ
→ Низкий: цепочка поставок NVLink-switch ASIC ограничивает поставки rack-scale конфигов первой половиной 2026 — механизм: дефицит у TSMC CoWoS-L.
→ Базовый: VAR-партнёры в РФ/СНГ получают доступ только к узловым SKU (HGX), не к полной стойке — механизм: экспортный контроль на интегрированные NVLink-домены свыше 72 GPU.
→ Высокий: появляются «совместимые» rack-scale решения от Supermicro/Wiwynn с конфигурируемой плотностью CPU 64–256 — механизм: OEM-конкуренция вынуждает NVIDIA открывать референс-дизайны через MGX.

Название: Добавить класс rack-scale конфигураций в BOM-генератор и граф совместимости
Триггер: NVIDIA Vera Rubin 256-CPU rack-scale (The Register, март 2026)
Затронутые компоненты: BOM-генератор, граф совместимости, TCO-калькулятор
Требования:
  1. Ввести в граф совместимости отдельный тип узла «rack-domain» с атрибутами NVLink-switch ревизии и количество CPU/GPU — без этого нельзя корректно валидировать конфигурацию выше 72 GPU.
  2. Добавить в TCO модель «мощность на стойку» с нелинейной зависимостью от плотности (CDU, шинопровод, ИБП) — иначе оценка для rack-scale будет заниженной на 20–40% [оценка автора].
  3. В BOM ввести флаг «экспортно-ограниченный SKU» с автоматическим предупреждением для B2G-сценариев — без этого риск выпуска нереализуемых спецификаций.
  4. Документировать референс-схемы подключения CDU и шинопровода для 200+ kW/стойка — требуется интеграторам на этапе проектирования.
Приоритет: P1 (в ближайший квартал)
Открытые вопросы блокирующие реализацию: п.1 (мощность/CDU), п.3 (экспортный контроль) из раздела выше.

✍️ ЧЕРНОВИК ПОСТА
В rack-scale конфигурации с 256 CPU в одной стойке исчезает привычное разделение «узел/стойка»: стойка становится единицей проектирования, а не сборки. Это меняет роль интегратора. Раньше VAR выбирал шасси и наполнял его картами; теперь объект поставки — преднастроенный домен с фиксированной топологией NVLink, фиксированным контуром охлаждения и фиксированной электрической нагрузкой.

NVIDIA подтвердила Vera Rubin rack-scale с 256 Vera CPU и прямым жидкостным охлаждением (The Register, март 2026). При таком уровне интеграции BOM перестаёт быть списком компонентов — это спецификация инженерной системы целиком: подвод 200+ kW, CDU, шинопровод, сейсмика, сервисный коридор.

Вывод для дистрибуции: либо платформа описывает rack-domain как первичную сущность с собственными ограничениями, либо она перестаёт быть применимой к новому классу поставок.

🔹 ТЕМА 2/4 · OPEN COMPUTE / CO-PACKAGED OPTICS
Lightmatter предлагает OCP-референс для co-packaged optics в AI-системах

📌 ФАКТ
Lightmatter подал в Open Compute Project референс-архитектуру для co-packaged optics (CPO), ориентированную на интеграцию оптических интерфейсов в корпус AI-ускорителя (HPCwire, 16 марта 2026). Детальные параметры (пропускная способность на линк, энергопотребление pJ/bit) в открытой части анонса не приведены.

📍 КОНТЕКСТ
До этого CPO продвигался преимущественно вендорами коммутаторов (Broadcom Tomahawk 5-Bailly, NVIDIA Quantum-X Photonics — анонсы 2024–2025). Перенос фокуса CPO на сторону ускорителя означает попытку устранить pluggable-оптику между NIC и switch на уровне топологии rack-to-rack.

🎯 РЕЛЕВАНТНОСТЬ ДЛЯ AI DISTRIBUTION PLATFORM
Средняя. CPO влияет на граф совместимости (новый класс интерфейса, не взаимозаменяемый с QSFP-DD) и на документацию (сервисная модель для оптики внутри ускорителя меняется). Прямого влияния на текущий BOM нет до появления коммерческих SKU.

🔮 ПРОГНОЗ НА 12 МЕСЯЦЕВ
→ Низкий: референс остаётся на уровне спецификации, коммерческих изделий в каталогах OEM нет — механизм: отсутствие согласованного MSA по оптическому коннектору между Lightmatter, Ayar Labs и Celestial AI.
→ Базовый: появляются демо-стенды у 1–2 OEM (Supermicro, Wiwynn) на OCP Summit 2026, но в продакшен не уходят — механизм: незрелость supply chain по лазерным источникам.
→ Высокий: гиперскейлер (Meta или Microsoft) анонсирует кастомный ускоритель с CPO на базе Lightmatter в качестве первичного клиента — механизм: внутренняя метрика энергии на бит в fabric перестаёт удовлетворять при переходе к 1.6T линкам.

3. Кто из ASIC-вендоров (Broadcom, Marvell, кастомные) поддержит интеграцию на уровне SerDes?

✍️ ЧЕРНОВИК ПОСТА
Co-packaged optics обсуждают преимущественно как способ снизить энергию на бит. Это правильный, но не единственный угол. С точки зрения эксплуатации CPO — это смена модели сервисного обслуживания: оптика перестаёт быть hot-swappable компонентом и становится частью неразборного модуля ускорителя.

Lightmatter передал в OCP референс-архитектуру CPO для AI-систем (HPCwire, март 2026). Если архитектура примется как стандарт, граф совместимости в дистрибуции перестаёт включать «трансивер QSFP-DD X» как отдельную позицию для соответствующего класса узлов — позиция растворяется внутри SKU ускорителя.

Вывод: для дистрибутора это означает смещение маржи и сервисной выручки. Замена трансивера за $2k уходит в RMA целого модуля за $30k+.
